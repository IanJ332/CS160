Project 2: Ticket Master

Overview

This project was developed during CS157A (Database Management Systems). The main objective was to create a ticketing website that provides functionalities such as publishing events, deleting events, modifying event details, and adding information like pictures, time, location. It also supports automatic redirection to Google Maps for event locations.

Key Features

  • Users can make purchases, request refunds, bind payment information, rate events, and write reviews.
  • Basic functionalities for event management include adding, editing, deleting, and viewing events.
  • Google Maps integration for location-based navigation.

Performance Optimizations

Significant efforts were made to optimize the backend database for better performance under heavy workloads. Advanced database concepts like optimistic locking and pessimistic locking were implemented to ensure data consistency and prevent logic errors in the system, even under high throughput.
