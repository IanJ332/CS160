Project 1: Local Jira

Overview

During CS151 (Object-Oriented Design), I developed a software similar to Jira. The primary purpose of this project was to enable individuals to track their projects more efficiently by synchronizing previous unfinished code. We implemented various tags such as development, fixes, and updates to streamline the workflow.

Additionally, we designed an interface that integrates with Eclipse. This interface allows users to:

  • Select a piece of code.
  • Create a ticket for it directly from the interface.
  • Reopen the software later, click on the ticket, and jump to the specific code section needing modification.
  • Automatically launch Eclipse software for code editing.

Main Features

  • Quick Ticket Creation: Create tickets that link to specific code sections, enabling quick access and modification.
  • To-Do List: Add new to-do items, create notes within the list, and keep track of tasks effectively.
  • Local Time Tracking: Automatically display the user's current local time and track personal timelines for project progress.
  • Encrypted Database: Securely store and manage project information using database encryption.

Technical Highlights

This project was built using a combination of Java for the backend logic and database encryption techniques to safeguard project data. The integration with Eclipse was achieved through plugin development, making it user-friendly for developers working in that environment.
